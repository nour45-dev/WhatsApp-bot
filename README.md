# بوت واتساب ذكي لمواعيد المدرسين — مجاني بالكامل (من غير AI)

نسختين من نفس البوت، تقدر تشغل أي واحدة فيهم:
- **QR (whatsapp-web.js)**: بتتصل برقمك الحالي على واتساب بيزنس عن طريق مسح كود، مجانية 100%، مناسبة للتجربة والاستخدام اليومي البسيط.
- **Meta Cloud API**: الطريقة الرسمية من ميتا، محتاجة توثيق شركة ورقم مخصص، لكنها أضمن للمدى الطويل.

⚠️ **الرقم الواحد بيشتغل بطريقة واحدة بس في نفس الوقت** — لو حولت رقم لـ Meta API هيتشال تلقائيًا من واتساب بيزنس العادي.

## البوت بيعمل إيه بالظبط
- بيدور على اسم الدكتور/المادة في جدول Google Sheets ويرد بالمعاد.
- **قائمة رئيسية** أول ما تكتب "اهلا" أو "قائمة": بحث بالمدرس / بالمادة / بالصف / عرض الجدول كامل.
  - في نسخة **Meta**: قائمة تفاعلية حقيقية تتدوس عليها (List Message رسمية من واتساب).
  - في نسخة **QR**: قائمة كنص مرقّم (1، 2، 3...) لأن الأزرار الحقيقية بقت مش شغالة في المكتبة الغير رسمية من فترة (مشكلة معروفة في whatsapp-web.js نفسها، مش حاجة تقدر تتصلح من عندنا).
  - في الحالتين، تقدر تكتب سؤالك مباشرة في أي وقت من غير ما تمر بالقائمة.
- **لو الدكتور بيدرّس أكتر من صف/سنة دراسية**، البوت بيسأل "انت في صف كام؟"، وبعدين "يوم إيه؟" لو لسه فيه أكتر من خيار، لحد ما يوصل لمعاد واحد بالظبط.
- **ذكاء اصطناعي احتياطي (اختياري ومجاني)**: لو حطيت `OPENROUTER_API_KEY` في `.env`، البوت هيجرب يسأل موديل مجاني عبر OpenRouter قبل ما يحول العميل للإدارة، وده بس لما محرك القواعد العادي معرفش يرد (مش كل رسالة) عشان نوفر من حد الطلبات المجاني.
- بيعرض النتيجة بشكل احترافي (أيقونات لكل حقل)، وبيخفي أي عمود فاضي.
- بيفتكر آخر حاجة اتكلمتوا فيها لمدة 15 دقيقة.
- بيرد طبيعي على "شكرا" و"السلام عليكم" و"مع السلامة".
- لو مش لاقي حاجة (حتى بعد محاولة الذكاء الاصطناعي)، بيحول العميل على الإدارة.

## تفعيل الذكاء الاصطناعي الاحتياطي (اختياري)
1. سجّل في https://openrouter.ai بإيميلك (من غير فيزا).
2. من لوحة التحكم، اعمل API Key جديد وانسخه.
3. حطه في `.env`:
   ```
   OPENROUTER_API_KEY=sk-or-...
   ```
4. البوت هيستخدم موديل مجاني تلقائيًا (`openrouter/free` بيختار أقوى موديل متاح مجانًا وقتها).
5. الحد المجاني بيكون تقريبًا 20 طلب/دقيقة و50-1000 طلب/يوم (بيتغير من وقت لتاني حسب OpenRouter) - كافي جدًا لأنه بيتفعل بس لما القواعد العادية تفشل.

⚠️ سيب `OPENROUTER_API_KEY` فاضي لو مش عايز الميزة دي، البوت هيشتغل عادي بالقواعد بس.

```
│   ├── sheetService.js        # جلب وتحديث بيانات الشيت

├── qr-version/

│   └── server.js               # صفحة عرض QR + health check
├── meta-api-version/
│   ├── index.js                # نسخة Meta Cloud API (webhook)

```

# 1) نسخة QR (whatsapp-web.js)

### التشغيل المحلي
```bash
npm install
cp .env.example .env
npm run start:qr
```
هيظهرلك QR في الـ Terminal، أو افتح `http://localhost:3000` في المتصفح.
من موبايل واتساب بيزنس: **الإعدادات > الأجهزة المرتبطة > ربط جهاز** وامسح الكود.

### النشر على Render/Railway
1. ارفع المشروع على GitHub.
2. على المنصة، اختار "Deploy from Dockerfile" وحدد `Dockerfile.qr`.
3. ضيف متغيرات البيئة من `.env.example` (القسم المشترك بس).
4. **فعّل Persistent Disk/Volume واربطه بمسار `/app/session`** (وإلا هتضطر تمسح QR كل مرة يعيد تشغيل).
5. بعد الـ Deploy، افتح رابط السيرفر، هتلاقي صفحة فيها QR، امسحها.

# 2) نسخة Meta Cloud API (الرسمية)

### الخطوة 1: إنشاء تطبيق على Meta for Developers
1. ادخل https://developers.facebook.com وسجل دخول بحساب فيسبوك بتاع الشركة.
2. My Apps > Create App > اختار نوع "Business".
3. من لوحة التطبيق، دوس "Add Product" ودور على **WhatsApp** وضيفه.
4. هتوصل لصفحة "API Setup" وفيها:
   - **رقم تجريبي مؤقت** من ميتا تقدر تبعت بيه لحد 5 أرقام تجربة (مفيش داعي لرقم حقيقي في الأول).
   - **Temporary Access Token** (صالح 24 ساعة، كويس للتجربة).
   - **Phone Number ID** (رقم ID مش رقم الموبايل).

### الخطوة 2: تجهيز الـ .env
```bash
cp .env.example .env
```
افتح `.env` واملأ:
```
META_VERIFY_TOKEN=اي كلمة سر تختارها انت (مثلاً: mySecret123)
META_ACCESS_TOKEN=(الصق التوكن من صفحة API Setup)
META_PHONE_NUMBER_ID=(الصق الـ Phone Number ID)
```

### الخطوة 3: تشغيل البوت وجعله مرئي للإنترنت
لازم Meta توصل لسيرفرك بـ HTTPS، فمحليًا محتاج أداة زي **ngrok** للتجربة:
```bash
npm run start:meta
# في نافذة تانية:
ngrok http 3000
```
انسخ الرابط اللي ngrok هيديهولك (زي `https://xxxx.ngrok-free.app`).

للنشر الحقيقي على Render/Railway (رابط ثابت من غير ngrok):
1. ارفع المشروع على GitHub.
2. اختار "Deploy from Dockerfile" وحدد `Dockerfile.meta`.
3. ضيف كل متغيرات `.env` في إعدادات المنصة (Environment Variables).
4. بعد الـ Deploy، هتاخد رابط ثابت زي `https://your-app.onrender.com`.

### الخطوة 4: ربط الـ Webhook على Meta
1. رجع لصفحة التطبيق على Meta for Developers > WhatsApp > Configuration.
2. في خانة **Callback URL** حط: `https://رابط-سيرفرك/webhook`
3. في خانة **Verify Token** حط بالظبط نفس القيمة اللي كتبتها في `META_VERIFY_TOKEN`.
4. دوس **Verify and Save** — لو ظهر ✅ يبقى نجح.
5. تحت "Webhook fields"، دوس **Subscribe** على حقل **messages**.

### الخطوة 5: التجربة
ابعت رسالة من أي رقم مسجل في قائمة "To" التجريبية (لو لسه مش وثقت رقم حقيقي)، وشوف البوت بيرد.

### للانتقال من التوكن المؤقت للدائم (بعد التجربة)
التوكن المؤقت بينتهي كل 24 ساعة. للاستخدام الحقيقي:
1. Meta Business Suite > إعدادات > "System Users" > أنشئ System User جديد.
2. اديله صلاحية على تطبيق الـ WhatsApp بتاعك.
3. ولّد Token دائم من هناك وحطه بدل القديم في `META_ACCESS_TOKEN`.
(الخطوات بتتغير بين فترة وأخرى حسب تحديثات ميتا، فراجع توثيقهم الرسمي وقت التنفيذ.)

## أمثلة على تعاملات هيفهمها البوت
- "معاد دكتور أحمد إمتى؟" → لو الدكتور بيدرّس صف واحد بس، يرد بالمعاد على طول.
- لو بيدرّس أكتر من صف: يسأل "انت في صف كام؟" → ترد "تالته ثانوي" أو "3ث" → يرد بالمعاد الصح.
- "الساعة كام؟" (بعد ما يرد على سؤال سابق) → يرد من الذاكرة من غير ما تعيد اسم الدكتور.
- "شكرا" / "السلام عليكم" / "مع السلامة" → ردود طبيعية.
- "الجدول كله" → كل المواعيد (لو مش طويل قوي).

## إزاي أطور الكود لاحقًا؟
كل المنطق الأساسي في `shared/matchEngine.js` — نفس الملف بيشتغل مع النسختين، فأي تعديل هناك بينعكس على الاتنين تلقائيًا:
- **الردود/الصياغات**: عدّل مصفوفات `GREETING_REPLIES`, `FALLBACK_REPLIES`, `MATCH_INTROS`.
- **تسميات وأيقونات العرض**: عدّل `LABEL_RULES`.
- **كلمات متابعة جديدة**: عدّل `FOLLOW_UP_FIELDS`.
- **كلمات الصف/السنة الدراسية**: عدّل `GRADE_HEADER_KEYWORDS` و `ORDINAL_TO_DIGIT`.
- **تغيير مدة تحديث الشيت**: من `SHEET_REFRESH_MINUTES` في `.env`.
- **الرد داخل الجروبات (نسخة QR)**: عدّل `shouldIgnore()` في `qr-version/index.js`.
